# rdf-shuttle

**Shuttle** is a grammar formalism for RDF concrete syntaxes: a deterministic
L-attributed translation grammar with relational semantics and lens-derived
printing. You write **one specification** of a syntax (Turtle, TriG,
N-Triples, N-Quads, SHACL Compact Syntax, N3) and three artifacts are derived
from it mechanically:

- a **streaming parser** — text to quads, single pass, quads emitted at the
  earliest point their arguments are ground, `O(depth)` memory;
- a **serializer** — the *backward reading* of the same productions: every
  `emit` clause becomes a linear consuming query over a residual multiset of
  quads, so serializer completeness is a checked property, and the parser and
  writer can never drift apart because there is no second spec;
- a **conformance test suite** — (document, expected-quads) pairs generated by
  running the grammar with nothing ground, coverage-directed and semantically
  valid by construction, plus *provably* negative syntax tests from the LL
  parse tables, emitted as a W3C-format `manifest.ttl`.

## The problem

Every RDF syntax today has at least three independent encodings of the same
language: a parser, a serializer, and a test suite (plus the prose spec they
all approximate). They drift. The parser and writer disagree on corner cases;
the test suite covers whatever its authors thought of; round-tripping is a
hope validated by whichever bugs users happen to hit.

Shuttle collapses the three into modes of a single relation

```
R_p ⊆ Tok* × Tok* × Env × Env × Bag(Quad) × ⟦params⟧ × ⟦value⟧
```

with **parse** (text ground), **print** (quads ground), and **generate**
(nothing ground) derived from one set of productions. Round-trip laws —
`parse(print(G)) ≅ G`, `parse ∘ print ∘ parse = parse`, print totality per
grammar — are compile-time obligations with counterexamples on failure, not
integration-test folklore.

## Why now: RDF 1.2

RDF 1.2 adds triple terms `<<( s p o )>>`, reified-triple sugar
`<< s p o ~ r >>`, annotation blocks `{| … |}` with a pending-reifier rule,
directional language tags `"…"@ar--rtl`, and a `VERSION` directive. Every
parser, writer, and test suite in the ecosystem must be extended — by hand,
against a spec still in Working Draft flux.

The pending-reifier semantics alone is subtle enough that coverage-directed
sampling of Shuttle's `annotation` production immediately exposed **three
distinct conformance failures** in a current, widely-used hand-written parser
(a trailing bare reifier duplicating the assertion, dropping the
`rdf:reifies` quad, and leaking the reifier into the next statement). That is
the argument in one sentence: when the syntax moves, hand-written triples of
(parser, writer, tests) tear; a generated triple cannot.

Shuttle's term algebra is RDF 1.2-complete and *positionally typed* — a triple
term in subject position, or `@lang^^datatype`, is a grammar-level type error,
not a runtime check.

## Status

v0.1 draft. The specification is under review; no compiler (`shuc`) exists
yet. Planned bootstrap order: hand-written front-end → N-Triples →
Turtle 1.2 → W3C-suite trust anchor → TriG / SHACL-C → N3, with self-hosting
of the meta-grammar as the v1 exit criterion. See
[`spec/SHUTTLE.md` §10](spec/SHUTTLE.md) for resolved and open design
questions.
